# 技能创建手册

## 技能概要

技能是以模块化形式扩展Claude能力的独立功能包，通过提供以下内容实现专业化赋能：
1. 专项工作流 - 特定领域的多步骤操作流程
2. 工具集成 - 处理特定文件格式或API的标准化方案
3. 领域知识 - 企业专用数据模式与业务逻辑
4. 资源包 - 复杂重复任务的配套脚本与素材

## 核心设计原则

### 语境经济性原则
技能内容需与系统提示、对话历史等其他内容共享有限的上下文窗口。设计要求：
- 仅包含Claude不具备的必需信息
- 优先采用精简示例而非冗长说明
- 每个新增内容需通过价值评估："这些token消耗是否必要？"

### 自由度调控原则
根据任务特性设置不同级别的操作规范：
▌高度自由（文字说明）
适用于多解问题或依赖情境判断的场景

▌中度自由（参数化脚本）
存在优选方案但允许适度调整的情况

▌低度自由（固定脚本）
关键操作流程或容错率极低的任务

类比说明：如同Claude在路径探索中，悬崖窄桥需严密护栏（低自由），而开阔原野可自由穿行（高自由）。

## 技能文件架构

标准技能包包含必需文件和可选目录：

```
技能名称/
├── SKILL.md（必需文件）
│   ├── YAML元数据（必需）
│   │   ├── 名称: （必填）
│   │   └── 描述: （必填）
│   └── Markdown使用指南（必需）
└── 资源目录（可选）
    ├── scripts/         - 可执行脚本
    ├── references/      - 按需加载的参考资料
    └── assets/          - 输出用素材文件
```

### SKILL.md规范
▌前端元数据（YAML格式）
必须包含`name`和`description`字段。这两个字段是Claude判断是否调用该技能的唯一依据，需精确描述：
- 技能核心功能
- 具体触发场景
（示例见后文【最佳实践】部分）

▌主体内容（Markdown格式）
仅在技能触发后加载，应避免包含"何时使用"类说明。

### 资源目录规范
▌脚本目录（scripts/）
存放需反复调用的可靠代码：
- 适用场景：确定性操作或高频重现代码
- 典型示例：PDF旋转脚本
- 优势：节省token，保证输出一致性

▌参考目录（references/）
存放按需调用的专业资料：
- 数据库结构文档
- API接口规范
- 企业政策文件
- 最佳实践指南
※ 超过1万字建议添加grep检索模式

▌素材目录（assets/）
存放产出物模板：
- 品牌视觉素材
- PPT/文档模板
- 前端组件库

### 禁用内容
技能包应避免包含：
- README等说明文档
- 安装指南
- 变更记录
等非直接支持功能的附属文件

## 渐进式披露设计

采用三级加载机制优化上下文使用：

1. 元数据层（常驻，约100词）
2. SKILL.md主体（触发后加载，<5000词）
3. 资源文件（按需调用）

### 内容组织模式
推荐三种结构化方案：

① 核心指南+扩展参照
```markdown
# PDF处理
基础操作：[基础示例]
高级功能：
- 表单填写 → 详见[FORMS.md]
- API参考 → 详见[REFERENCE.md]
```

② 领域细分结构
```
bigquery-skill/
├── SKILL.md（总览）
└── reference/
    ├── 财务指标.md
    ├── 销售数据.md
    └── 产品分析.md
```

③ 条件化细节
```markdown
## 文档编辑
基础操作：[示例]
修订跟踪 → 详见[REDLINING.md]
OOXML详解 → 详见[OOXML.md]
```

※ 关键要求：
- 参照文件不超过两级嵌套
- 超百行文件需包含目录概要

## 开发流程

### 阶段1：需求定位
通过具体用例明确技能边界，例如图像编辑技能需确认：
- 支持的功能范围（旋转/去红眼等）
- 典型用户问法
- 潜在扩展需求

### 阶段2：资源规划
分析每个用例，确定需要封装的资源：
- 高频重现代码 → 放入scripts/
- 领域知识文档 → 放入references/
- 产出物模板 → 放入assets/

### 阶段3：技能初始化
运行自动化脚本生成标准化框架：
```bash
scripts/init_skill.py <技能名称> --path <输出目录>
```

### 阶段4：内容开发
▌元数据编写范例：
```yaml
name: docx专业处理
description: |
  支持Word文档全生命周期管理，包括：
  1. 新建标准化文档
  2. 内容修订与批注
  3. 修订跟踪处理
  4. 格式规范检查
```

▌主体内容要求：
- 采用命令式/不定式语态
- 复杂流程分步骤说明
- 每个脚本备注使用场景

### 阶段5：质量验证
执行打包命令时自动检测：
```bash
scripts/package_skill.py <技能路径> ./dist
```
校验项包括：
- 元数据完整性
- 目录结构合规性
- 文件引用有效性

### 阶段6：迭代优化
根据实际使用反馈：
1. 记录操作瓶颈点
2. 优化说明文档
3. 补充典型用例
4. 更新资源文件
